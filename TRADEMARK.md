# Trademark policy

The name **ModelMeld**, the **ModelMeld** logo, and any related visual
identity (collectively, "the Marks") are trademarks of ModelMeld.
This policy explains how the Marks may and may not be used.

The AGPL-3.0-or-later license that governs the source code in this repository
**does not grant any rights to the Marks**. Trademark rights are
separate from copyright. You can fork the code freely under AGPL-3.0-or-later;
you cannot use the Marks to brand your fork as if it were ours.

## Permitted uses (no permission needed)

- **Referring to the project** in articles, blog posts, talks,
  comparisons, books, courseware. "ModelMeld is an AI gateway that..."
  is fine. Use the Marks accurately and in the same form as we do
  (capitalization, spacing).
- **Linking back to us** in documentation that integrates with
  ModelMeld. "Configure your client to point at your ModelMeld
  instance" is fine.
- **Stating compatibility.** "Works with ModelMeld" or "Compatible
  with ModelMeld" is fine as long as the claim is accurate.
- **Personal projects, education, research.** Use the Marks
  descriptively in non-commercial settings.
- **Bug reports, security disclosures, community participation.** No
  permission needed; that's the whole point of an open community.

## Restricted uses (permission required)

- **Branding a competing hosted service.** If you operate a service
  that competes with ModelMeld's commercial offering (the curated
  registry feed, the managed hosted tier), you may not call your
  service "ModelMeld," "powered by ModelMeld," or use the Marks in a
  way that implies endorsement.
- **Merchandise.** T-shirts, mugs, stickers, etc. bearing the Marks
  require permission.
- **Modified logos.** If you want to display a customized version of
  the ModelMeld logo (e.g. for a conference talk or article header),
  please ask first to make sure the modification respects brand
  consistency.
- **Domain names + social handles.** Don't register domains or
  handles that incorporate "ModelMeld" in ways that could create
  confusion about whether they're official (e.g.
  `modelmeld-enterprise.com`, `@modelmeld-official`). The official
  domain is `modelmeld.ai`.

## Forks

You are welcome to fork the code under AGPL-3.0-or-later. When you do:

- **Rename your fork** if you publish it under a different brand or
  intend to operate it as a distinct product. "ModelMeld" as a fork
  name implies the project endorses your version, which we cannot
  do for forks.
- **Keep attribution.** AGPL-3.0-or-later requires preserving copyright
  notices and the NOTICE file; we additionally request that you
  reference upstream prominently in your README so users understand
  the relationship.

## Examples

- ✅ "I'm using ModelMeld to route my Claude Code traffic." (descriptive,
  accurate)
- ✅ "My open-source project integrates with ModelMeld." (descriptive)
- ✅ Blog post titled "Why I migrated from LiteLLM to ModelMeld."
  (commentary, comparison)
- ✅ A fork named `acme-gateway` based on modelmeld, with attribution
  to ModelMeld in the README. (renamed fork, attribution preserved)
- ❌ A SaaS named "ModelMeld Cloud" that competes with our hosted tier.
- ❌ Logo with the cat-meld glyph swapped for a different mascot,
  published as "modified ModelMeld logo."
- ❌ Domain `modelmeld-pro.com` registered by a third party.

## Questions + permission requests

If you're unsure whether a use is permitted, ask. Email
`hello@modelmeld.ai` with the specifics; we typically respond within
a week. Most well-intentioned uses get a quick "yes, go ahead";
restricted uses get a discussion about whether a license arrangement
makes sense.

## This policy can change

We'll update this file as new uses come up. Material changes go through
a GitHub Discussion before merging so the community can weigh in. Older
uses that became permitted under a prior version are honored
indefinitely (we don't apply trademark policies retroactively to
existing users).
